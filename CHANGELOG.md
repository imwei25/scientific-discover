## 未发布

### 数据分析：提升任务成功率（T2 探索轮 / T3 方法路由 / T4 修复循环与红线）

- **探索轮（T2）**：正式写分析代码前，先让 AI 写一段“只读探索代码”在真实数据上跑一遍，把
  真实观测（列是否存在、分组到底几组各组多少例、连续变量分布）喂回代码生成——从根上消灭
  “臆测列名/结构”这一数据分析头号失败源。探索失败自动跳过，不影响后续。
- **确定性方法路由 + 决策卡（T3）**：新增 `statroute.py`。AI 只做“把研究问题拆成分析规格”
  的语义抽取，**由代码在真实数据上跑前提检验（正态性 Shapiro / 方差齐性 Levene / 期望频数）
  并用决策树选统计方法**（t 检验 / Welch / Mann-Whitney / ANOVA / Kruskal / 卡方 / Fisher /
  相关…），再把选定方法喂回代码生成。这样 AI 没机会“前提没满足却硬跑参数检验”。选定方案以
  “决策卡”展示（数据情况 / 前提检查 / 建议方法 / 备选），也是你核对“分组有没有搞反”的关口。
- **更聪明的自动修复（T4）**：修复时不再只喂报错——附上真实数据诊断（`df.shape`、各列 dtype），
  并要求模型“先说错因再改、可改用 scipy/statsmodels 绕开库问题”；若在**同一个错误上反复失败**，
  自动切换为“换一种思路/库从头重写”（而非继续贴补丁）。
- **产品红线（T4）**：结论区固定提示“AI 生成、正式使用前请由专业统计人员核对；显著≠临床意义”，
  结论措辞也要求审慎（统计显著不等于因果/临床意义）。

### 数据分析：提升任务成功率（T1 确定性快赢）

- **统一数据读取**：子进程执行不再自行重读原始文件，改由主进程用统一的健壮解码
  （`textio.read_csv_bytes`）读入并去重列名后、以 pickle 传给子进程。彻底消除“AI 看到的
  列名”与“执行时真实列名”因两套解码不一致而导致的 KeyError（本该成功却失败）。
- **更聪明的数据画像**：画像新增数值列描述统计、分类列主要取值(计数)，并对每列做启发式
  体检——自动标注“看着是文本其实是数字”（含千分位/百分号/货币/删失阈值 `<`）、疑似日期列、
  疑似 ID/编号列、未被识别的缺失标记（`-`/`缺失`/`999` 等），提示 AI 先清洗再分析。
- **结果确定性体检**：分析跑通后用规则（不调用 AI）核对输出——`p=0.000` 应写为 `p<0.001`、
  p 值越界、缺少必需的【方法选择】/【假设检查】/【数据质量】区块——并把问题喂回结论环节让
  AI 据实修正或说明。
- **修复：危险调用护栏误杀**——扫描前先剥离字符串与注释，图表标题/print 文本里出现的
  `open(`、`eval(` 等不再被误判而整段拒绝执行（真实的系统/网络调用仍拦截）。
- **修复：结论输入截断丢数字**——结论所依据的真实输出过长时改为保留头尾（尾部常含主分析
  结果/p 值），不再一刀切截断前 6000 字。
- **修复：统计顾问结果卡崩溃**——前端按后端契约（`recommended={test,why}`、
  `alternatives=[{test,when}]`）渲染对象，不再因把对象当 React 子节点而报错。
