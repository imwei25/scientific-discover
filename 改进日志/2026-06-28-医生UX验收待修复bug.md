## 🔴 P1 必修

### Bug-1：统计顾问完全不渲染（W1-6 功能报废）

- **症状**：数据分析 → 切到"📚 统计顾问" Tab → 输入问题（如"我有 60 人分两组比较 HbA1c"）→ 提交 → 按钮恢复正常 → **下方四张推荐卡片永远不出现** → 没有任何错误提示
- **根因**：前后端 SSE 协议错位
  - **后端** `backend/app/main.py` 的 `/api/stats/advice` 路由：把 LLM 原始 token 流逐片发出，每片是 `data: {"text": "```"}` `data: {"text": "json"}` ...
  - **前端** `frontend/src/modules/AnalyzeModule.tsx` 约 1061-1076 行：对每个 SSE event 做 `JSON.parse(raw) as AdvisorPayload`，期望每个 event 就是一个完整结构化 payload → 每次都抛异常 → `lastValid` 永远是 `{}` → `hasResult=false` → UI 空白
  - 异常被 catch 住没冒到 console，所以**没有任何外部信号**
- **修复方向（任选其一）**：
  - **A（推荐 · 改前端）**：累加所有 `text` 片段拼成完整字符串 → 等 SSE `done` 事件 → 剥掉 ```json``` 围栏 → 一次性 `JSON.parse` → setLastValid
  - **B（改后端）**：在 LLM 流式完毕后追发一个独立事件 `event: result\ndata: <完整 JSON>`，前端只在收到 `result` 事件时解析
- **回归测试建议**：在 `backend/test_main.py`（或新建 `test_stats_advice_e2e.py`）加一个用 mock LLM 模拟 token 流的端到端用例，确保前端能拼出完整 JSON
- **截图**：`.playwright-mcp/wave3/step10_advisor_state.png`

## 🟡 P2 体验问题（建议一起修）

### Bug-2：左侧 9 个导航没用 Lucide 图标

- **症状**：左侧 sidebar 的 nav-item 只显示 "01-09" 数字编号 + 文字标题；首页 home-card 已经用了 Lucide，**两处不一致**。折叠侧栏后只剩 24px ticks，**完全无法识别模块**
- **spec 来源**：Wave 2 设计文档 D5"emoji icons 一次性替换为 Lucide 单色图标 · 左侧导航 8 个 emoji → Lucide 对应"
- **根因**：W2 agent 改了首页 home-card 的图标，但**漏改了 sidebar 的 NAV 渲染**。NAV 数组里的 `icon` 字段被绑给了首页卡片，sidebar 那边只用了 `nav-num`（编号）
- **修复方向**：在 `frontend/src/App.tsx` 的 sidebar nav 渲染处（约 130-145 行 `pipeline` div 内），把 nav-item 的内容加上对应 Lucide 图标（参考首页 `home-card-icon` 的渲染方式）。折叠态时只显示图标 + tooltip
- **截图**：`.playwright-mcp/wave3/step02_home_after_clear.png`

### Bug-3：同步角标错误态不切换

- **症状**：人为让 `PUT /api/state` 返回 500 持续 8 秒后，sidebar 底部的 `sync-badge` 仍停在"… 保存中"（petrol 色），没切换到 red error 态
- **影响**：网络中断或后端崩溃时医生**完全不知道修改没保存**，可能丢稿子
- **CSS 已就位**：`.sync-badge.error { background: var(--bad-soft); color: var(--bad); font-weight: 600 }` 定义齐全
- **根因**：状态机问题——失败 catch 后是否真的 `setSyncStatus('error')`？是否被后续的 saving setter 覆盖？需排查
- **修复方向**：检查 `frontend/src/lib/projects.ts`（或 `useProjects` hook 里同步逻辑）的 catch 分支，确认 `setSyncStatus('error')` 被调用且不被并发请求的 saving 状态盖掉
- **截图对比**：`.playwright-mcp/wave3/step17_after_type.png` vs `.playwright-mcp/wave3/step17_sync_error.png`

### Bug-4：找选题"引用核验"识别面太窄

- **症状**：报告底部显示"✓ 引用核验：正文 0 处文献引用均来自本次检索到的真实文献"，但实际正文里有大量 `[Mao et al., 2026]` `[Zhong et al., 2025]` 这种纯文本引用
- **影响**：医生看到"0 处"会怀疑引用是不是 AI 编的，但事实是校验程序没识别这种格式
- **修复方向**：`backend/app/citations.py`（或 verify 相关逻辑）的引用提取正则补一条：识别 `\[[A-Z][a-zA-Z\- ]+\s+(?:et al\.,?\s+)?(\d{4}[a-z]?)\]` 这种纯文本引用，而不只 markdown 链接 `[text](url)`
- **截图**：`.playwright-mcp/wave3/step07_search_result.png`

## 🟢 P3 小瑕疵（可选）

### Bug-5：品牌图标和模块页标题仍用 emoji

- 各模块标题 `💡 找选题` / `📊 数据分析与写作` / `📋 伦理材料` 等

- **建议**：sidebar 底部加一行"按 `Ctrl+K` 快速跳转"，或 Help 模态里加快捷键页

### Bug-7：`/favicon.ico` 404
- 页面图标缺失，浏览器标签页是空白图标
- **建议**：放一个 16/32 px 的实验室/分子图到 `frontend/public/favicon.ico`

### Bug-8：localStorage `clear()` 不能"重置体验"
- 因为 backend project state 立即覆盖回来
- **建议**（仅开发者调试）：加个 `?reset=1` query 参数触发"清 state + 清 localStorage + 跳过 health 自动加载"
