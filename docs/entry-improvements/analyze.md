# 入口：数据分析与写作 (analyze)

上传数据(csv/excel) → 本地统计分析 + 出图(PNG/SVG/PDF) → AI 提炼核心观点、写论文段落
（数字由本地 Python 算，不让 LLM 算数）。含 general / 森林图 / KM / ROC 多种分析。
代码：`frontend/src/modules/AnalyzeModule.tsx`；样式：`frontend/src/styles.css`。

> 注：本模块近期刚被重点强化过「成功率」（探索轮/方法路由/修复循环），本轮聚焦界面/交互/可用性层面。

---

## 第 1 圈 · 2026-07-02

### 起因（批判要点）
派 agent 从「非 IT 医学/药学研究者第一次分析自己的数据」视角批判，12 条，按严重度排序。摘录：

**高**
1. 上传后**完全没有数据预览**，用户无法确认文件读对了（合并单元格/多余表头/乱码），只能盲点「开始分析」，信任崩塌。
2. **xlsx + KM/ROC 是死路**：`ColMapper` 在无表头时提示「请直接输入列名」，但组件里只有 `<select>`、**根本没有输入框**，也没有运行按钮——功能彻底走不通，而 xlsx 是医学最常见格式。
3. 「真实计算结果」只有折叠的原始 stdout（等宽 `<pre>`），没有可读的统计结果表，把最有信服力的证据藏了起来。

**中**
4. 「分析类型」下拉切换整块改变界面，且森林图根本不用上传的文件（`runForest` 只用手填表），心智模型跳变。
5. 结果工具栏按钮被 `.result-toolbar` 的 `space-between` 拉散（3 个按钮直接平铺，未用 `.result-actions` 包裹），空隙大而不均、观感松散。
6. 结论区没有「一键复制」，医学用户最想要的「粘进 Word/发给导师」动作缺失。
7. 刷新后残留旧结果但 `file` 不持久化 → 显示完整结论却因 `disabled={!file}` 无法重跑，错乱。
8. 长耗时分析只有一行状态文字，无阶段/进度/耗时。
9. 报错一行红字 + 原始技术信息、无重试。

**低**
10. forest/km/roc 结果无免责声明、统计量(I²/Log-rank p/AUC…)零解释。
11. 无示例数据/模板/默认问题，冷启动门槛高。
12. 拖拽区太单薄（细 bar），不像可拖放区域。

### 本轮取舍
以 **#2（xlsx 死路，最高优先的可走通性 bug）** 为核心，顺带两个低风险打磨：#5 工具栏编组、#6 复制结论（与 idea 一致）。数据预览(#1)、可读统计表(#3)、分阶段进度(#8) 等较大改动留待后续轮次。

### 改动
- `AnalyzeModule.tsx` `ColMapper`（#2）— 无表头（xlsx 等）时不再是死路：把每个列映射从「只有 `<select>`」退化为**可手动输入列名的 `<input>`**，并保留运行按钮 + 一句「需与首行列名完全一致」的提示。因 `ColField` 的 value/onChange 与 select 路径共用同一状态，KM/ROC 运行逻辑无需改动，零后端/契约变更。
- `AnalyzeModule.tsx` 结论工具栏（#5）— 把「用此结论去排版 / 导出HTML / 导出MD」三个按钮用 `<div className="result-actions">` 包裹，右对齐编组，不再被 `space-between` 拉散。
- `AnalyzeModule.tsx` 结论工具栏（#6）— 新增「复制结论」按钮：一键复制结论到剪贴板，短暂显示「已复制 ✓」/失败态；用局部 `copyState` 状态，不触碰会带 spinner 的 `status`。

### 预期效果
- xlsx 用户也能做 KM/ROC 分析（手动输入列名），补上一条彻底走不通的路径。
- 结论工具栏按钮编组紧凑、观感精致。
- 用户可一键复制结论去粘贴，省掉下载文件或手动划选。
- `npm run build` 通过（tsc + vite，无报错）。

### 未做 / 留待后续轮次（analyze 第 2 圈优先）
- **#1（高）**：上传后加数据预览（行列数 + 前几行 + 字段类型），让用户确认文件读对了。
- **#3（高）**：把原始 stdout 解析/呈现为可读的统计结果表（这是「可信」卖点的关键证据）。
- **#7**：`file` 缺失但有旧结果时给「结果对应的数据已丢失，请重新上传」提示，或清理陈旧结果。
- **#8** 分阶段进度（探索→生成→执行→修复）；**#9** 报错友好化 + 重试；**#4** 森林图不吃上传文件的跳变；**#10** 统计量 tooltip + 全路径免责；**#11** 示例数据；**#12** 更醒目的拖放区。
