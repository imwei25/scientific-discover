# 入口：期刊排版 (format)

按目标期刊要求重排稿件并导出 Word；含参考文献格式化/核验、投稿包、LaTeX 导出。
可从上一步带入 `format:manuscript` / `format:refs` / `format:journal`。
代码：`frontend/src/modules/FormatModule.tsx`；后端：`backend/app/formatting.py`、`journals.py`；样式：`frontend/src/styles.css`。

---

## 第 1 圈 · 2026-07-02

### 起因（批判要点）
派 agent 从「非 IT 医学/药学研究者第一次排版投稿」视角批判，并与已打磨模块做一致性对比。先取证：期刊库只有 5 条预设（general_en/general_cn/nature/ieee/plos_one），对临床/药学作者实际只有「通用英文 IMRaD」「中文核心 GB/T 7714」两条有用——这是多条问题的根因。12 条，摘录：

**高**
1. **期刊库覆盖太小且无「找不到目标刊」出口**：原生 `<select>` 无搜索/自由输入，只有 5 条预设。用户投的 Lancet/JAMA/中华医学系列根本没有；header 却承诺「按该刊的结构与格式要求重排」，与实际能力严重不符——信任杀手。
2. 整页一长条并列操作，无推荐顺序/步骤引导（重排→下载 Word→LaTeX→投稿包→参考文献格式化/核验），第一次用不知先后。

**中**
3. 期刊默认静默选中第一项（general_en），中文作者没注意会把中文稿按英文 IMRaD 排。
4. 必填/禁用无解释，按钮变灰不说为什么。
5. Diff 弹窗「接受/拒绝」后果不透明，拒绝会 `setText("")` 连带清掉「下载 Word」入口。
6. 就绪检查、参考文献核验结果没有复制/导出（与全局一致性背离）。
7. 所有报错都无「重试」，且是原始技术文案。
8. 「清空」一键抹掉全部（含整篇稿件），无二次确认。
9. **「下载 Word」是否含参考文献取决于隐藏前置动作**：只有先到下方点「按该期刊格式化参考文献」，Word 才带参考文献；默认下载没有，用户以为丢了。
10. 参考文献区信息密度高、「核验」与「格式化」两回事没说清；口径过度承诺「格式准确」（GB/T 7714 在 citeproc 下中英混排本有 i18n 缺陷，见 [[format-feature-gbt-csl-finding]]）。

**低**
11. 稿件/参考文献输入框对「整篇论文」太小（rows 8/5）。 12. Header 塞满 LaTeX/IEEEtran/Overleaf 术语，淹没核心价值。

### 本轮取舍
最高的 #1（期刊库覆盖）需后端扩期刊库/映射，是大改；但它真正的伤害是**信任落差**（over-promise + 无出口）。本轮用纯前端、零行为变更的**诚实化 + 首用引导**三连，把用户最先撞上的困惑消掉：

### 改动
- `FormatModule.tsx`（#1/#3）— 目标期刊下拉下方加**覆盖说明**：点明「期刊库目前只内置少数通用/示例模板；没有你的目标刊？先选通用英文/中文核心模板作基础，投稿前对照目标刊官网 Author Guidelines 微调」。给「找不到目标刊」一个明确出口，并校正 header 的 over-promise。
- `FormatModule.tsx`（#4）— 稿件为空时在按钮下方加门槛提示「请先粘贴稿件正文（或上传 Word/PDF）才能重排」，禁用按钮自解释。
- `FormatModule.tsx`（#9）— 「下载 Word」下方在**尚未格式化参考文献**时加提示：「下载的 Word 暂不含参考文献；如需带上，请到下方参考文献区点『格式化』后再下载」，解开「正文重排 vs 参考文献」两块的隐藏耦合。

### 预期效果
- 用户不再误以为下拉里就该有自己的目标刊、也知道没有时怎么办，header 的过度承诺被如实补正。
- 灰按钮点不动时知道缺什么。
- 下载出的 Word 没有参考文献不再让人以为「丢了」，而是知道要先格式化。
- `npm run build` 通过（tsc + vite，无报错）。
