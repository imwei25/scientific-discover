# 入口：首页 (home)

首页 = 用户打开应用第一眼看到的落地页：hero（品牌标题 + 副标 + CTA + 手绘牛马装饰）+ 下方 9 个工具卡片总览。
代码：`frontend/src/App.tsx` 的 `Home`/`Niuma3D`/`NiumaMark`；样式：`frontend/src/styles.css`。

---

## 第 1 圈 · 2026-07-02

### 起因（批判要点）
派 agent 从「非 IT 的医学/药学科研人员第一次打开」视角批判，列出 12 条，按严重度排序。摘录关键：

**高**
1. 主标题「niuma-research」（等宽小写英文代号）用户看不懂是什么产品，也没有一句中文说明它能做什么。
2. eyebrow + 副标题全是氛围文案，不说明功能。
3. 真正的 9 个工具入口在首屏之下，需滚动才能看到（笔记本 + 125% 缩放下 hero 几乎占满一屏）。

**中**
4. 半屏、持续漂浮的卡通牛马装饰喧宾夺主。
5. 唯一 CTA 把所有人塞进「找选题」，已有数据/初稿的用户没有入口。
6. 卡片编号 01–09 + 「完整工作流」暗示必须按顺序走完，实际上 9 个工具相互独立、可单独使用。
7. 9 张卡片密度高、单色线性图标区分度低，难扫读。

**低（含代码卫生）**
11. 遗留/冲突 CSS：死类 `.home-art-wrap`/`.hero-svg`（旧「分子骨架」样式，已不再渲染）；`.home .home-grid { animation: none }` 指向根本不存在的类（真实类名是 `.home-cards`，且它本就无动画）；`.home .eyebrow/h1/home-sub` 与更具体的 `.home-text …` 规则重复（被 specificity 覆盖，属冗余死码）。

### 本轮取舍
高/中里的第 1、2、4 条与用户**近期刻意的品牌决策**冲突——「niuma-research / 牛马」标识与手绘吉祥物是最近两次 commit（444ab42、7a19243）专门做的，hero 文案也刚被用户手工精简过（「去小字」）。故本轮**不动品牌与 hero 文案**，只做两处 **brand-safe** 的真实改进：

### 改动
- `App.tsx:450` — 概览区 eyebrow 文案：`完整工作流 · 从想法到投稿` → `覆盖选题到投稿 · 每个工具都能单独用`。直接消解第 6 条「必须按顺序走完」的误解，不触碰品牌。
- `styles.css` — 删除死码 `.home-art-wrap` / `.hero-svg` / `[data-theme=midnight] .hero-svg`（旧分子骨架样式，DOM 里已无对应类）。
- `styles.css` — 删除失效/冗余的 `.home .eyebrow/h1/home-sub` 与指向空类的 `.home .home-grid { animation:none }`；替换为一条作用明确、保留概览 eyebrow 淡入的 `.home-overview .eyebrow { animation: page-in … }`。hero 的 eyebrow/h1/sub 仍由更具体的 `.home-text …` 规则负责。

### 预期效果
- 新用户不再误以为 9 个工具是必须逐步走完的流水线，降低「我只想做数据分析是不是得先做前 4 步」的困惑。
- 首页 CSS 去掉一段死码 + 一条指向不存在类的规则，减少后续改版踩坑概率、提升可维护性（注：`.home h1` 双重规则原本已由 specificity 解决，非用户可见闪烁，此处是卫生清理）。
- `npm run build` 通过（tsc + vite，无报错）。

### 未做 / 留待后续轮次
- 第 1/2/3 条（标题看不懂、功能不明、入口在首屏下）需与用户确认对品牌的态度后再动——可考虑在**不改标题**的前提下，于 hero 加一句中文「一句话说明能做什么」，或压低 hero 高度让卡片上浮。
- 第 5 条（多入口 CTA）、第 7 条（卡片可扫读性/图标区分度）留待下一圈。
