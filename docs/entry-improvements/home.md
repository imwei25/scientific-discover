# 入口：首页 (home)

首页 = 用户打开应用第一眼看到的落地页：hero（品牌标题 + 副标 + CTA + 手绘牛马装饰）+ 下方 9 个工具卡片总览。
代码：`frontend/src/App.tsx` 的 `Home`/`Niuma3D`/`NiumaMark`；样式：`frontend/src/styles.css`。

---

## 第 1 圈 · 2026-07-02

### 起因（批判要点）
派 agent 从「非 IT 的医学/药学科研人员第一次打开」视角批判，列出 12 条，按严重度排序。摘录关键：

**高**
1. 主标题「niuma-research」（等宽小写英文代号）用户看不懂是什么产品，也没有一句中文说明它能做什么。
2. eyebrow + 副标题全是氛围文案，不说明功能。
3. 真正的 9 个工具入口在首屏之下，需滚动才能看到（笔记本 + 125% 缩放下 hero 几乎占满一屏）。

**中**
4. 半屏、持续漂浮的卡通牛马装饰喧宾夺主。
5. 唯一 CTA 把所有人塞进「找选题」，已有数据/初稿的用户没有入口。
6. 卡片编号 01–09 + 「完整工作流」暗示必须按顺序走完，实际上 9 个工具相互独立、可单独使用。
7. 9 张卡片密度高、单色线性图标区分度低，难扫读。

**低（含代码卫生）**
11. 遗留/冲突 CSS：死类 `.home-art-wrap`/`.hero-svg`（旧「分子骨架」样式，已不再渲染）；`.home .home-grid { animation: none }` 指向根本不存在的类（真实类名是 `.home-cards`，且它本就无动画）；`.home .eyebrow/h1/home-sub` 与更具体的 `.home-text …` 规则重复（被 specificity 覆盖，属冗余死码）。

### 本轮取舍
高/中里的第 1、2、4 条与用户**近期刻意的品牌决策**冲突——「niuma-research / 牛马」标识与手绘吉祥物是最近两次 commit（444ab42、7a19243）专门做的，hero 文案也刚被用户手工精简过（「去小字」）。故本轮**不动品牌与 hero 文案**，只做两处 **brand-safe** 的真实改进：

### 改动
- `App.tsx:450` — 概览区 eyebrow 文案：`完整工作流 · 从想法到投稿` → `覆盖选题到投稿 · 每个工具都能单独用`。直接消解第 6 条「必须按顺序走完」的误解，不触碰品牌。
- `styles.css` — 删除死码 `.home-art-wrap` / `.hero-svg` / `[data-theme=midnight] .hero-svg`（旧分子骨架样式，DOM 里已无对应类）。
- `styles.css` — 删除失效/冗余的 `.home .eyebrow/h1/home-sub` 与指向空类的 `.home .home-grid { animation:none }`；替换为一条作用明确、保留概览 eyebrow 淡入的 `.home-overview .eyebrow { animation: page-in … }`。hero 的 eyebrow/h1/sub 仍由更具体的 `.home-text …` 规则负责。

### 预期效果
- 新用户不再误以为 9 个工具是必须逐步走完的流水线，降低「我只想做数据分析是不是得先做前 4 步」的困惑。
- 首页 CSS 去掉一段死码 + 一条指向不存在类的规则，减少后续改版踩坑概率、提升可维护性（注：`.home h1` 双重规则原本已由 specificity 解决，非用户可见闪烁，此处是卫生清理）。
- `npm run build` 通过（tsc + vite，无报错）。

### 未做 / 留待后续轮次
- 第 1/2/3 条（标题看不懂、功能不明、入口在首屏下）需与用户确认对品牌的态度后再动——可考虑在**不改标题**的前提下，于 hero 加一句中文「一句话说明能做什么」，或压低 hero 高度让卡片上浮。
- 第 5 条（多入口 CTA）、第 7 条（卡片可扫读性/图标区分度）留待下一圈。

## 第 2 圈 · 2026-07-02

### 起因（批判要点，聚焦第 1 轮未覆盖的新问题）
派 agent 二轮评审（明确不再动品牌/吉祥物/hero 文案）。8 条，摘录：

**高**
1. 卡片左上角大号衬线序号 01–09 仍是页面视觉权重最高的数字元素，强暗示「必须按顺序走」，与第 1 轮改的 eyebrow「每个工具都能单独用」自相矛盾（文案说独立、视觉说顺序）。
2. 九张卡片读起来像「说明性概览/工作流图解」而非可点入口——affordance 只藏在 `cursor:pointer` 和 hover 里，触屏/键盘用户看不到「这能点」；首屏注意力被暖色 hero CTA 独占。

**中**
3. `.home-meta`（唯一说明「给谁用」的定位文案）用 `--faint`(#93a09d) + 12px + 0.12em 字距，在 `--paper` 上对比度约 2.6:1，未过 WCAG AA(4.5:1)。
4. 九个图标里五个是「带横线的文档」变体，同尺寸同粗细同色，扫一眼分不清写标书/论文初稿/期刊排版。
5. 宽屏 `auto-fill minmax(230)` 算出 4 列 → 9 张排成 4/4/1，末行「孤儿卡」视觉失衡。

**低**
6. 卡片入场 stagger 只给了 nth-child(1)~(4) delay，后 5 张一起蹦出，是硬编码 4 张时代的残留。
7. 卡片序号未 `aria-hidden`，读屏会朗读「01/02…」噪音（而图标反而已隐藏，处理不一致）。
8. 遗留误导注释 `/* hero art —— 分子骨架 + 拖拽旋转 */`（现在是牛马）；`.home-card-num` 硬编码 Georgia 字体栈不走 `--serif` token；`.home-card-icon` 残留对 SVG 无效的 `font-size:18px`。

### 本轮取舍
品牌约束下不动序号本身（它是 nav/首页统一的设计语言）；改为用**箭头入口提示**把卡片的「读法」从「第 N 步」扭向「点此进入 →」，同时修一批 a11y/精致度问题。图标区分度(#4，需逐卡配色)留待后续。

### 改动
- `App.tsx` 卡片（#2）— 每张卡尾部加 `→`（`.home-card-go`，`aria-hidden`）：常驻可见（不靠 hover），让触屏/键盘用户也一眼看出可点；hover 时变亮并右移 3px。
- `App.tsx` 卡片（#7）— 给 `.home-card-num` 加 `aria-hidden="true"`，读屏不再朗读装饰序号。
- `styles.css`（#5）— `.home-cards` 加 `@media(min-width:900px){ grid-template-columns: repeat(3,1fr) }`：宽屏固定 3 列，9 张排成齐整的 3/3/3，消除孤儿卡；窄屏仍回落 auto-fill。
- `styles.css`（#3）— `.home-meta` 颜色 `--faint`→`--muted`、字距 0.12em→0.06em，过 AA、更易读。
- `styles.css`（#6）— stagger 补到 nth-child(1)~(9)，九张卡依次淡入。
- `styles.css`（#8）— 修 stale 注释为「手绘 3D 牛马装饰」；`.home-card-num` 字体改走 `var(--serif)`；删 `.home-card-icon` 对 SVG 无效的 `font-size`。

### 预期效果
- 卡片一眼看出「可点进入」，触屏/键盘用户不再漏掉九个工具入口；箭头也把「第 N 步」的读法冲淡。
- 定位文案「面向医学·药学·生物医学研究者」看得清了。
- 宽屏卡片排列齐整、入场动效连贯、读屏无序号噪音、CSS 去掉一批漂移。
- `npm run build` 通过。

### 未做 / 留待后续轮次（home 第 3 圈优先）
- **#4**：给九个工具图标做区分（换更具象的图标或分组配色），降低「五个文档变体」的辨识负担。
- #1 序号是否保留/弱化，涉及与 nav 一致的设计语言，需与用户确认后再定。
