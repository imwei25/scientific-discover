# 入口：找选题 (idea)

用户输入研究方向/关键词 → 检索真实文献 → 现状/空白矩阵 + 有文献支撑的候选课题。
代码：`frontend/src/modules/IdeaModule.tsx`；样式：`frontend/src/styles.css`；检索白名单佐证：`backend/app/searchfilters.py`。

---

## 第 1 圈 · 2026-07-02

### 起因（批判要点）
派 agent 从「非 IT 医学/药学科研人员第一次用」视角批判，12 条，按严重度排序。摘录：

**高**
1. 默认就把检索限死在「综述/RCT/Meta/系统综述」四种出版类型（`studyTypes` 全选，藏在默认折叠的高级设置里），会静默剔除队列/病例对照/横断面/机制/临床前研究 → 用户莫名「结果很少/为空」而不自知。
2. 点「开始」后不直接检索，先插入两道 AI 问答卡（聚焦方向 / 方向优化），期间只有按钮文字变化、无 spinner/进度 → 像卡死。

**中**
3. 结果区关键下一步（做实验规划 / 写标书）被做成最弱的 ghost 按钮，且 `result-actions` 无 `flex-wrap`，一行挤 5–6 个按钮会横向溢出/截断。
4. 全程没有「复制」按钮——医学用户最自然的「复制这段进我的 Word」缺失，只能下载整个 .docx。
5. 通用报错甩后端英文技术串，且除 PubMed 零命中外无「重试」入口。
6. 引用核验失败时未核验 URL 一串粘连、无分隔/编号/正文定位。
7. 文献质量筛选全是行话（SJR 分区 / 自造「影响力」），定义只藏在 hover tooltip。
8. 取消全部论文源后按钮变灰，但原因提示藏在折叠面板里。
9. 追问区按钮层级与危险度相反：「按此修改报告」（会覆盖整篇）是弱 ghost，「追问」反而是强 primary。

**低**
10. 占位文案说「填好左侧信息」，但表单在上方非左侧。
11. `.result-status` 对中文用了 `text-transform:uppercase`（对 CJK 无效）+ 0.08em 字距，排版发虚。
12. 起始年份只有近 5/7/10 年预设，无自定义。

### 本轮取舍
挑「小而可落地、低风险、真实收益」的三条先做；#1、#2 属于会改变检索/交互行为的产品判断，留待后续轮次并在下方标注。

### 改动
- `IdeaModule.tsx` — **新增「复制」按钮**（#4）：一键把调研报告（含参考文献）复制到剪贴板，成功后按钮短暂显示「已复制 ✓」（1.8s）；剪贴板未授权时回落到 `status-line` 文案提示。新增 `copied` 局部状态，不用会带 spinner 的 `setStatus` 做成功反馈。
- `styles.css` `.result-actions` — 加 `flex-wrap: wrap` + `justify-content: flex-end`（#3a）：按钮多时自动换行右对齐，不再横向溢出/截断。
- `styles.css` `.result-status` — 去掉对 CJK 无效的 `text-transform:uppercase`，字距 `0.08em → 0.02em`，加 `white-space:nowrap`（#11）：中文状态文字更紧凑、换行时不被挤散。

### 预期效果
- 用户可直接复制整篇报告去粘进标书/论文，省掉「为拿一段字下载整份 Word」的成本。
- 窄画布下结果工具栏按钮换行而非溢出，动作不再被截断。
- 状态文字排版更精致、无冗余无效声明。
- `npm run build` 通过（tsc + vite，无报错）。

### 未做 / 留待后续轮次（idea 第 2 圈优先）
- **#1（最高）**：隐藏的默认出版类型白名单导致结果失真——建议在主区加可见提示「当前仅检索综述/RCT，点此放宽」，或把默认改为不限类型（需产品决策，涉及检索行为变更）。
- **#2**：点开始→两道问答卡期间加 spinner/进度提示，或说明「正在帮你聚焦方向」。
- #3b：把「做实验规划 / 写标书」这类价值转化按钮提到更强的视觉层级。
- #5 报错友好化 + 通用重试；#6 未核验链接分隔编号；#7 行话就地解释；#9 追问区按钮层级纠偏。
