# 入口：找选题 (idea)

用户输入研究方向/关键词 → 检索真实文献 → 现状/空白矩阵 + 有文献支撑的候选课题。
代码：`frontend/src/modules/IdeaModule.tsx`；样式：`frontend/src/styles.css`；检索白名单佐证：`backend/app/searchfilters.py`。

---

## 第 1 圈 · 2026-07-02

### 起因（批判要点）
派 agent 从「非 IT 医学/药学科研人员第一次用」视角批判，12 条，按严重度排序。摘录：

**高**
1. 默认就把检索限死在「综述/RCT/Meta/系统综述」四种出版类型（`studyTypes` 全选，藏在默认折叠的高级设置里），会静默剔除队列/病例对照/横断面/机制/临床前研究 → 用户莫名「结果很少/为空」而不自知。
2. 点「开始」后不直接检索，先插入两道 AI 问答卡（聚焦方向 / 方向优化），期间只有按钮文字变化、无 spinner/进度 → 像卡死。

**中**
3. 结果区关键下一步（做实验规划 / 写标书）被做成最弱的 ghost 按钮，且 `result-actions` 无 `flex-wrap`，一行挤 5–6 个按钮会横向溢出/截断。
4. 全程没有「复制」按钮——医学用户最自然的「复制这段进我的 Word」缺失，只能下载整个 .docx。
5. 通用报错甩后端英文技术串，且除 PubMed 零命中外无「重试」入口。
6. 引用核验失败时未核验 URL 一串粘连、无分隔/编号/正文定位。
7. 文献质量筛选全是行话（SJR 分区 / 自造「影响力」），定义只藏在 hover tooltip。
8. 取消全部论文源后按钮变灰，但原因提示藏在折叠面板里。
9. 追问区按钮层级与危险度相反：「按此修改报告」（会覆盖整篇）是弱 ghost，「追问」反而是强 primary。

**低**
10. 占位文案说「填好左侧信息」，但表单在上方非左侧。
11. `.result-status` 对中文用了 `text-transform:uppercase`（对 CJK 无效）+ 0.08em 字距，排版发虚。
12. 起始年份只有近 5/7/10 年预设，无自定义。

### 本轮取舍
挑「小而可落地、低风险、真实收益」的三条先做；#1、#2 属于会改变检索/交互行为的产品判断，留待后续轮次并在下方标注。

### 改动
- `IdeaModule.tsx` — **新增「复制」按钮**（#4）：一键把调研报告（含参考文献）复制到剪贴板，成功后按钮短暂显示「已复制 ✓」（1.8s）；剪贴板未授权时回落到 `status-line` 文案提示。新增 `copied` 局部状态，不用会带 spinner 的 `setStatus` 做成功反馈。
- `styles.css` `.result-actions` — 加 `flex-wrap: wrap` + `justify-content: flex-end`（#3a）：按钮多时自动换行右对齐，不再横向溢出/截断。
- `styles.css` `.result-status` — 去掉对 CJK 无效的 `text-transform:uppercase`，字距 `0.08em → 0.02em`，加 `white-space:nowrap`（#11）：中文状态文字更紧凑、换行时不被挤散。

### 预期效果
- 用户可直接复制整篇报告去粘进标书/论文，省掉「为拿一段字下载整份 Word」的成本。
- 窄画布下结果工具栏按钮换行而非溢出，动作不再被截断。
- 状态文字排版更精致、无冗余无效声明。
- `npm run build` 通过（tsc + vite，无报错）。

### 未做 / 留待后续轮次（idea 第 2 圈优先）
- **#1（最高）**：隐藏的默认出版类型白名单导致结果失真——建议在主区加可见提示「当前仅检索综述/RCT，点此放宽」，或把默认改为不限类型（需产品决策，涉及检索行为变更）。
- **#2**：点开始→两道问答卡期间加 spinner/进度提示，或说明「正在帮你聚焦方向」。
- #3b：把「做实验规划 / 写标书」这类价值转化按钮提到更强的视觉层级。
- #5 报错友好化 + 通用重试；#6 未核验链接分隔编号；#7 行话就地解释；#9 追问区按钮层级纠偏。

## 第 2 圈 · 2026-07-02

### 起因（批判要点）
派 agent 二轮评审，重点核实第 1 轮遗留高优先项并找新问题。核实/新增，摘录：

**严重**
1. **默认检索被静默限制（核实成立）**：`studyTypes` 初始值仍是全选 [rct/meta/systematic/review]（`IdeaModule.tsx:125-128`），UI 藏在默认折叠的高级设置 `<details className="adv-settings">`（无 `open`，L507）里；后端 `searchfilters.py:137-156` 把它拼成排他白名单 `AND (...)`，把队列/病例对照/横断面/机制研究全过滤掉；主按钮附近无任何可见提示（`label()` 只进 `status`，而 status 仅检索中才渲染）。
3. **【新·严重 bug】clarify/refine 两次预检请求无 try/catch**：`onStart`(L264-276)/`goRefine`(L279-302) 是裸 await，一旦网络/500/超时 reject，`setClarifying(false)`/`setRefining(false)` 永不执行 → 主按钮 `disabled` 含 `clarifying||refining`，**永久禁用并停在「聚焦方向中…」**，界面无任何错误，只能刷新整个应用。
4. **【新·严重】证据等级复选框语义反直觉**：`_types_clause` 空列表=不过滤、非空=白名单。于是「四项全勾（默认）=结果最受限，四项全不勾=不限」，与「全勾=全都要」的普遍预期完全相反；且选项里根本没有队列/病例对照/横断面可选，hint 也没说这条隐藏规则。

**高—中**
2. **点开始后两道 AI 问答卡期间无 spinner/进度（核实成立）**：clarify/refine 从不 `setStatus`，status-line 不渲染，用户只见禁用按钮文字变化，误以为卡死。
5. 限制性过滤用 `usePersistentState` 会跨不相关课题静默延续，折叠态无「已改动 N 项」提示。
6. 「清空」无二次确认，会连同已生成报告一起抹掉。

**低**
7. 证据等级过滤跨源行为不一致（PubMed/EPMC 严筛、OpenAlex 近似、Crossref 完全不落地），混在同一列表用户无从得知哪些被筛。

### 本轮取舍
优先修 **#3（会锁死核心入口的真实 bug）+ #2（同区域的进度反馈缺失）**，二者同在 onStart/goRefine；再顺带 **#4/#1/#7 的就地诚实化**（改 hint 说清反直觉语义，零行为变更、不牵动「改默认」的产品决策）。#1 改默认、#5 折叠徽标、#6 清空确认留待后续。

### 改动
- `IdeaModule.tsx`（#3+#2）— `onStart`/`goRefine` 的 await 套 `try/catch/finally`：`finally` 复位 `clarifying`/`refining`（不再锁死）；进入时 `setStatus("正在为你聚焦研究方向…"/"正在优化研究方向…")` 复用 status-line 给出 spinner 反馈，弹卡时清掉；**catch 里回退到直接 `submit()`**（预检本是可跳过的增强步骤，错误由 submit 的 onError 呈现）。
- `IdeaModule.tsx`（#4/#1/#7）— 证据等级 `filter-hint` 改写为讲清反直觉语义：「勾选后仅保留所勾类型；想纳入队列/病例对照/横断面/机制等观察性或基础研究，请取消全部勾选（=不限类型）」，并注明 OpenAlex 近似、Crossref 不按类型过滤。让隐藏白名单陷阱就地可见。

### 预期效果
- 一次瞬时网络抖动不再把「开始检索」永久锁死；预检失败自动回退到直接检索，用户无感或看到正常错误。
- 点开始→两道问答卡期间有「正在聚焦/优化方向…」spinner，不再像卡死。
- 用户看得懂「全勾=最受限、取消勾选=不限」，想纳入观察性研究知道该怎么做，减少静默结果偏倚。
- `npm run build` 通过。

### 未做 / 留待后续轮次（idea 第 3 圈优先）
- **#1 改默认**：把 `studyTypes` 默认改为 `[]`（不限），或主按钮下方常驻过滤摘要 chip「当前仅检索：综述/RCT…（点此调整）」——涉及检索行为变更，需产品决策。
- **#4 控件重做**：证据等级改成带「不限」总开关的显式控件，而非靠「取消全部」表达不限。
- **#5** 折叠 summary 加「已改动 N 项」徽标；**#6** 清空在有报告/文献时二次确认。
