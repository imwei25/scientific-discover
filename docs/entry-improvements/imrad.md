# 入口：论文初稿 (imrad)

把已有材料（背景/方法/结果/讨论/参考文献）装配成 IMRaD 结构的医学论文初稿 + 结构式摘要 + 关键词/MeSH
（只据真实材料、不编造）。代码：`frontend/src/modules/ImradModule.tsx`；样式：`frontend/src/styles.css`。

---

## 第 1 圈 · 2026-07-02

### 起因（批判要点）
派 agent 从「非 IT 医学/药学研究者第一次拼初稿」视角批判，并与已打磨模块（idea/analyze 等）做一致性对比，12 条，按严重度排序。摘录：

**高（均为一致性缺口）**
1. **生成完初稿后无法「进下一步」**：`App.tsx` 挂载 `<ImradModule />` 时**没有传 `goto`**（对比 idea/analyze 都传了），整份模块没有任何去选刊/去排版按钮。初稿正是通往「智能选刊/期刊排版」的天然节点，用户只能手动复制→切模块→再粘贴，容易以为「流程到此为止」。
2. **初稿/摘要/关键词三个结果面板全都没有「复制」按钮**（对比 idea 有带「已复制 ✓」反馈的复制）。医学作者最常见动作就是把某段（摘要/关键词）粘进投稿系统，关键词面板甚至连导出都没有。
3. 结构式摘要要重新粘一遍要点，不能从已生成初稿/结果带入（`genAbstract` 硬性要求 `absPoints`，无回退）。

**中**
4. 「从各模块导入」是静默操作，不知道导入了什么、也不知道没导到什么。
5. 字段全是「粘贴 XX」占位、无示例、无「至少填一项」门槛提示。
6. 「清空」一键抹掉全部素材+结果，无二次确认。
7. 目标字数是自由文本、乱填静默回落 250。
8. 页面极长、5 个大 textarea + 摘要区 + 投稿包信息密度失衡、无折叠分区。
9. 「推荐关键词」在无素材时点了没反应（静默 no-op）。

**低**
10. 出错后无「重试」。 11. PHI 检测开关是标题区裸 checkbox、层级怪。 12. 状态提示都是同档灰色小字、不消退、易漏看。

### 本轮取舍
选两个最高、且延续前几轮「一致性」主题的缺口：**#1（goto 交接，最大缺口）+ #2（复制按钮）**。二者一起把「初稿→选刊/排版」这条工作流真正接上，并补齐与其他模块一致的复制能力。摘要智能带入(#3)、导入反馈(#4)、清空确认(#6) 等留待后续轮次。

### 改动
- `App.tsx` + `ImradModule.tsx`（#1）— 给 `ImradModule` 传入并接收 `goto` prop（`import type { Goto }`）。初稿结果工具栏新增两个交接按钮：
  - 「用此初稿去选刊 →」→ `goto("journal", { "journal:abstract": abstract || draft })`（选刊按摘要匹配，无摘要时回退用初稿正文）。
  - 「用此初稿去排版 →」→ `goto("format", { "format:manuscript": draft, "format:refs": refs })`（把初稿与参考文献一并带入排版）。
  - localStorage 键经核对：format 读 `format:manuscript`/`format:refs`，journal 读 `journal:abstract`。
- `ImradModule.tsx`（#2）— 新增 `copiedKey` 状态 + `copyToClipboard(key, text)` 助手；在**初稿 / 摘要 / 关键词**三个工具栏各加「复制」按钮，成功短暂显示「已复制 ✓」。关键词面板此前连导出都没有，这补上了它唯一的取出方式。

### 预期效果
- 初稿生成后可一键带着它去「选刊」或「排版」，工作流不再断裂，用户不会以为流程到此为止。
- 摘要、关键词、初稿都能一键复制去粘贴到投稿系统/Word，补齐与 idea/analyze 一致的体验。
- `npm run build` 通过（tsc + vite，新 `goto` prop 类型对齐 App，无报错）。

### 未做 / 留待后续轮次（imrad 第 2 圈优先）

- **#4**：「从各模块导入」加成功/跳过反馈（导入了哪些、因已有内容跳过哪些）。
- **#9**：无素材时禁用「推荐关键词」或给提示，消除静默 no-op。
- **#5** 门槛提示 + 示例；**#6** 清空二次确认；**#7** 目标字数用 number/校验；**#8** 长页面折叠分区；**#11** PHI 开关提级。
