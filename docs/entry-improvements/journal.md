# 入口：智能选刊 (journal)

用摘要/标题匹配候选投稿期刊（OpenAlex 相似刊聚合）。代码：`frontend/src/modules/JournalMatchModule.tsx`；
后端：`backend/app/journalmatch.py`；样式：`frontend/src/styles.css`。

---

## 第 1 圈 · 2026-07-02

### 起因（批判要点）
派 agent 从「非 IT 医学/药学研究者第一次找投稿期刊」视角批判，并与已打磨模块做一致性对比。先取证：后端 `match_journals` 只产出 journal/count/is_oa/in_doaj/issn/samples/reason 六个字段，**没有分区/影响力/匹配度**；OpenAlex 只取 ≤50 篇、search 截断 600 字、按 count 排序取前 10。12 条，摘录：

**高**
1. 选刊最该看的**分区/影响力完全缺席**（后端就没算），而同项目找选题给每篇文献都挂了 Q1/影响力徽章。
2. **死胡同**：`App.tsx:373` 未传 `goto`，模块内无「去排版/去写作」交接；且 OpenAlex 自由文本刊名与排版模块基于固定 `journal_id` 的期刊库对不上，交接在数据层也没打通。
3. 唯一排序信号「相近文献 N 篇」不透明且统计脆弱（50 篇样本、个位数计数噪声大、第 1 名进度条永远 100% 像满分），无不确定性提示。
4. 刊名点开是 Google 搜索（国内打不开）而非官网/投稿页；采了 ISSN 却不用。

**中**
5. 无排序/筛选（只看 OA/DOAJ 都做不到）。 6. 结果不能复制/导出（与全项目惯例背离）。 7. 术语零解释（DOAJ/OA 无 tooltip、无 HelpButton）。 8. 无停止、无重试，且把原始异常类名甩给用户。 9. 无输入门槛提示；长摘要静默截断 600 字。

**低**
10. 匹配理由标注失败时整列静默消失、无兜底。 11. 徽章语义借用错位、结果未进 CanvasSlot、长英文样例密集不截断。 12. 无「已从上一步带入摘要」的可见反馈。

### 本轮取舍
最高的 #1（分区/影响力缺失）与 #2（去排版交接）都需要**后端/数据层改造**（算分区影响力、把自由文本刊名映射到排版期刊库），不适合「小而可控」的一轮。故本轮聚焦让这张很单薄的结果页**可用且诚实**、并补齐一致性小件，全部为纯前端加法、不改匹配行为：

### 改动
- `JournalMatchModule.tsx`（#6）— 结果区顶部加工具栏：「共 N 本候选」+「复制清单 / 导出 Markdown」。`hitsMarkdown()` 把候选拼成带序号、OA/DOAJ 标签、匹配理由与「仅供参考」脚注的 Markdown。复制带「已复制 ✓」反馈。
- `JournalMatchModule.tsx`（#7）— 给「开放获取」「DOAJ」徽章加 `title` 解释（OA=免费公开、通常需版面费；DOAJ=开放获取期刊目录，可作正规性参考）。
- `JournalMatchModule.tsx`（#8）— 报错框加「重试」按钮（调 `submit`），不再失败后只能回主按钮重来。
- `JournalMatchModule.tsx`（#3/#9）— 强化结果下方说明：点明「对最多约 50 篇相近文献聚合、非影响因子/分区排名、样本有限噪声大、条形图仅表示相对相近度而非质量/匹配度评分」，给用户正确的心理预期。

### 预期效果
- 候选刊单可一键复制/导出去和导师讨论、存档，补齐与其他模块一致的取出能力。
- 用户看得懂 OA/DOAJ 是什么、不再把条形图误当权威排名。
- 匹配失败可一键重试。
- `npm run build` 通过（tsc + vite，无报错）。
