# 科研助手 · 使用说明

这是一个帮你做科研的智能助手，覆盖**从想法到投稿、再到返修**的完整流程：
找选题 → 实验规划 → 数据分析 → 论文初稿 → 智能选刊 → 期刊排版 → 报告规范核对 → 回复审稿。
不需要懂电脑，照下面三步就能用。

---

## 第一步：安装（只需做一次）

1. 找到 `scripts` 文件夹里的 `setup.ps1` 文件。
2. **右键点它 → 选择「使用 PowerShell 运行」**。
3. 看到绿色的「安装完成！」字样就好了（第一次会下载一些东西，请耐心等几分钟）。

> 如果右键没有「使用 PowerShell 运行」，可以在文件夹空白处按住 Shift 右键 →「在终端中打开」，
> 然后输入：`powershell -ExecutionPolicy Bypass -File scripts/setup.ps1` 回车。

## 第二步：填一次 API 密钥

科研助手需要连接一个 AI 大模型才能工作。

1. 打开 `backend` 文件夹里的 `.env` 文件（用记事本即可）。
2. 把 `LLM_API_KEY=` 后面换成你自己的密钥。
   - 默认用的是 **DeepSeek**（便宜好用）。去 deepseek.com 注册可获得密钥。
   - 也可以换成硅基流动、OpenAI 或 Claude，文件里有注释说明怎么切换。
3. 保存关闭。

## 第三步：使用

以后每次使用，只要**双击 `启动科研助手.bat`**，它会等服务就绪后自动打开浏览器。
（如果服务已经在运行，再次双击会直接打开浏览器，不会重复启动，放心点。）

用完想彻底退出：**双击 `停止科研助手.bat`**（或关闭那个最小化的 “RA-Server” 窗口）。
注意：只关浏览器或只关黑色提示窗口，服务仍在后台运行——这是正常的，下次打开会自动复用。

## 各功能怎么用（左侧导航按科研流程从上到下排列）

| 功能 | 你要做的 |
|---|---|
| 💡 **找选题**（医/药/生物） | 填研究方向，**多源检索真实文献**（PubMed / Europe PMC / OpenAlex，可选在研临床试验），梳理研究现状与空白矩阵，给出有文献支撑的候选课题；引用可点开、自动核验真伪。还能一键**提取 PICO / 纳排标准**、对结果**追问或让 AI 按意见修改**。 |
| 🗺️ **实验规划** | 写下研究想法，生成规范研究方案；内置**样本量计算器**、**统计分析计划(SAP)** 生成、**随机化分组表**（可复现、导出 CSV），均可导出 Word。 |
| 📊 **数据分析与写作** | 上传数据（Excel/CSV），AI **写分析代码并在本地运行**，基于真实结果出图表与结论（数字都由代码算出）。图表可选 PNG/SVG/PDF、期刊配色，并能**生成规范图注**。 |
| 📝 **论文初稿** | 把已有材料（或"从各模块导入"）装配成 **IMRaD 初稿**（只据真实材料、不编造），生成**结构式摘要+字数核对**、**关键词/MeSH**，并能**一键打包投稿包 ZIP**。 |
| 🎯 **智能选刊** | 粘贴摘要，匹配适合投稿的**候选期刊**（OpenAlex 相似文献聚合，含是否开放获取与匹配理由）。 |
| 📄 **期刊排版** | 选目标期刊重排稿件、导出 Word；**参考文献按期刊规范格式化**（Vancouver/GB‑T 7714/IEEE 等）并**核验真伪/撤稿/去重**；一键**预提交体检 + 生成投稿信**。 |
| ✅ **报告规范核对** | 按 **STROBE/CONSORT/PRISMA/SPIRIT/ARRIVE** 逐条自查稿件；**statcheck 统计一致性自查**（本地重算 p 值）；生成 **PRISMA/CONSORT 流程图**。 |
| ✍️ **回复审稿** | 粘贴审稿意见，拆解后逐条生成 **point-by-point 回复信**（本地处理、数据不出网），导出 Markdown/Word。 |

> 💡 每个页面都支持**拖拽上传文件**：数据分析拖 Excel/CSV；其它页面可拖 Word/PDF/txt，系统自动读取文字。

> 🔗 **一键串联**：找选题→实验规划、数据分析→期刊排版均可一键带数据过去；"论文初稿"能从各模块导入已有成果、并把全部产出打包成投稿包 ZIP。所有结果都会自动存入**历史记录**，可随时恢复。

## 让局域网里其他人也能用（可选）

默认只有本机能用。如果想让同一个办公室/实验室网络里的其他电脑也能访问：

1. 打开 `backend\.env`，把 `HOST=127.0.0.1` 改成 `HOST=0.0.0.0`，保存。
2. 重新双击 `启动科研助手.bat`。启动后那个黑色窗口里会显示一行「局域网访问: http://192.168.x.x:8756」。
3. 同一网络的其他设备，在浏览器里打开那个 `http://192.168.x.x:8756` 即可使用。
4. 如果别人打不开，多半是 Windows 防火墙拦了入站端口。**双击 `scripts\允许局域网访问.bat`**（会弹出管理员授权，点"是"），它会自动放行 8756 端口并打印出局域网网址。

> ⚠️ 安全提醒：开启局域网后，同一网络里**任何人都能使用你的 API key 消耗额度**，也能使用数据分析（会在你这台电脑上运行代码）。仅在可信的内部网络使用，用完可改回 `HOST=127.0.0.1`。

## 常见问题

**问：双击启动后，浏览器没打开 / 一直起不来？**
答：先**双击根目录的「检查环境.bat」**做一次环境自检——它会逐项检查 Python、依赖、密钥、端口，并在有问题的行后用 `->` 给出修复办法。最新版的 `启动科研助手.bat` 在启动失败时也会自动跑这个自检。按提示修好再重试即可。

**问：提示端口被占用 / 换一个端口？**
答：可能是上次的服务还在后台（直接打开浏览器访问即可），或 8756 端口被别的软件占了。要换端口：打开 `backend\.env`，加一行 `PORT=8780`（或其它 1024–65535 的数字），保存后重新启动。

**问：左下角显示「本地服务未连接」？**
答：说明后台没启动。请确认你是通过双击 `启动科研助手.bat` 打开的，并且那个最小化的 “RA-Server” 窗口还开着。

**问：提示「余额不足」之类的错误？**
答：你的 AI 密钥额度用完了，去对应平台充值，或在 `.env` 里换一个密钥。也可以在 `.env` 配置 `FALLBACK_*` 备用供应商——主供应商额度用尽时会**自动切到备用**继续工作。

**问：找选题「未检索到相关文献」？**
答：多半是检索式太窄或用了不规范的中文术语。系统会给出 **AI 改写建议**，照着改方向/关键词再试；也可以**放宽年份或证据等级过滤**、换用更通用的英文关键词。

**问：数据分析报错 / 失败了？**
答：AI 写的统计代码偶尔首次会报错，系统会**自动修正并最多重试 3 次**。若仍失败，常见原因是数据列名混乱、空表或格式特殊——可整理表头（避免重复列名）、确认不是空文件后重试，或把研究目的写得更具体一些。支持 Excel/CSV，中文 Excel 导出的 GBK 编码也能读。

**问：引用旁边显示「未核验」是什么意思？**
答：找选题的每条引用都会**自动核验**链接是否对应真实检索到的文献。「已核验」表示该文献确实来自本次检索；「未核验」是提醒你这条引用没能对上检索结果，**请务必自己点开链接核对**后再使用，谨防个别幻觉引用。

**问：能换成更强 / 别的 AI 模型吗？**
答：可以。打开 `backend\.env`，按文件里的注释切换 `LLM_PROVIDER` / `LLM_MODEL` / `LLM_API_KEY`（支持 DeepSeek、硅基流动、OpenAI 兼容接口、Claude 等）。写作/找选题这类需要更强推理的场景，换更强的模型效果更好。

**问：数据会被上传到网上吗？**
答：数据分析（算统计、画图）完全在你自己的电脑上完成，不会上传。只有写作时会把**算好的统计结果**（不是原始数据）发给 AI 帮你润色成文。

**问：AI 写的内容能直接用吗？**
答：AI 是助手不是代笔。数字都由你的电脑真实计算，但文字观点请你务必自己核对后再使用。
